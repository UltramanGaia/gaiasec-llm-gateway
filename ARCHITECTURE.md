# LLM Gateway 架构设计

## 概述
LLM Gateway 是一个将多种 LLM API 接口聚合到 OpenAI 兼容接口的中间件服务，提供模型映射、凭证管理、请求路由和日志记录等功能。

## 系统架构

![LLM Gateway Architecture](https://i.imgur.com/placeholder.png)

### 核心组件
1. **API 层**：提供 OpenAI 兼容的 RESTful API 接口
2. **路由层**：根据模型名称映射到对应的 LLM 提供商
3. **凭证管理**：安全存储和管理各 LLM 提供商的 API 密钥
4. **模型映射**：维护用户自定义模型名称到实际模型的映射关系
5. **请求/响应处理**：处理请求转发和响应转换
6. **日志记录**：记录所有请求和响应信息到 SQLite 数据库
7. **前端界面**：提供配置管理和日志查看功能

## 技术栈
- 后端：Go 语言
- 前端：Vue 3 + Vite
- 数据库：SQLite
- ORM：GORM
- Web 框架：标准库 net/http

## 详细设计

### 1. 数据模型
已实现以下数据模型：
- `Provider`：LLM 提供商信息，包含名称、API 密钥和基础 URL
- `ModelMapping`：模型映射关系，包含别名、提供商 ID 和实际模型名称
- `RequestLog`：请求日志，包含用户令牌、模型名称、请求和响应内容

### 2. API 接口设计
- `/chat/completions`：OpenAI 兼容的聊天完成接口
- `/api/providers`：提供商管理接口
- `/api/model-mappings`：模型映射管理接口
- `/api/credentials`：凭证管理接口
- `/api/logs`：请求日志查询接口

### 3. 核心功能实现
- **请求路由**：根据模型名称查找对应的提供商和实际模型，转发请求
- **凭证安全**：加密存储 API 密钥，运行时解密使用
- **模型映射**：支持用户自定义模型别名
- **请求/响应转换**：将不同提供商的 API 格式转换为 OpenAI 兼容格式
- **日志记录**：记录所有请求和响应，支持按时间、模型、用户等维度查询

### 4. 前端界面设计
- **提供商管理**：添加、编辑、删除 LLM 提供商
- **模型映射**：配置模型别名和映射关系
- **凭证管理**：生成和管理访问凭证
- **日志查看**：查看请求响应历史，支持过滤和搜索
- **调用链展示**：可视化展示请求的调用链路
